Projects

"MTVDB" - 2022

Last update: 24/02/22

I created this project for two reasons: first to be able to learn how to consume an API, and use that information to develop a website; secondly, I love watching movies and series, and I'm always interested in looking for information about what I see.

This website provides detailed information about Movies, Series and TV shows, including the people who appear in them. It includes the possibility of filtering by categories ('Popular', 'Top Rated', etc.) and also has a built-in search engine.

Welcome Page

First view of the website.

Categories view

When filtering Movies or Series by category, pages with a maximum of 20 cards are displayed. It also indicates the number of results and pages of the category.

Categories view

At the bottom of every page, there is a 'Back to top' button.

Details view

Clicking on a card (of a movie or series) shows detailed information about it, including genres and the main actors/actresses. YouTube trailers are also included if available.

Details view

By clicking on the "Full cast..." button, you are directed to the section that shows you all the actors/actresses that appear in the movie or series, including the name of the character.

Actor/Actress view

The most important information about the actor/actress is detailed. The badge that indicates their age changes color depending on whether the person is alive or not.

Actor/Actress view

By clicking the "Movies and TV Shows with..." button, the list of credits of the person is displayed, ordered from the most recent to the oldest.

Search engine

From all views of the site you can access the search button in the navigation bar. From it, a box opens with the space to enter the searched keyword, and also includes the possibility of filtering the search by category (to obtain more exact results).

Search results with no filter

If a category is not selected when performing a search, the first 5 results in each category for the searched keyword are returned.

Search results with filter

In the case of choosing a filter, all the results for the searched keyword are listed.


The controller consumes "The Movie Database" API, which I found to be great for practice. It returns information with the JSON structure; I captured that same information using Spring's 'RestTemplate' in Java.

The project was made with the MVC pattern (Model View Controller), using Thymeleaf to generate reusable templates in the frontend. Once finished, I did the deploy with Railway.

Last but not least: the website is fully responsive. It adapts perfectly on mobile devices.

What did I use?

Backend

  • Java 8
  • Spring framework
  • TMDB API

Frontend

  • HTML5/CSS3
  • Bootstrap v5.1
  • Thymeleaf

Where can you check it out?

The website is deployed via Railway.

"Librería Egg" - 2021

Last update: 17/12/21

This project is an excercise from the Web Full Stack Developer course at Egg Educación institute.

It is a web management system for a library, where users can register and access every book available to apply for a loan. It also gives them the option to modify their profiles, and access a section where they can see records of every loan under their names (current and expired).

Login/register page

This is the first page everyone sees.

Users view

Homepage for regular users includes information like profile photo, user name and lastname, and register date.

Users view

Users also can see every book available to apply a loan.

Users view

Clicking "Know more" button displays full book's information, and lets the user apply to a loan.

User loans

This is the user's loans management section (active and old loans).

Administrator view

This role has management functions on users, books and loans.

User management dashboard

Displays all the information of every user registered, and also gives direct access to management functions (disable user, add a book loan for the user, view all loans of the user, change user's role, and delete user from database).

Books management dashboard

Displays all the information of every book registered, and also gives direct access to management functions (disable book, modify book information and delete book from database).

Loans management dashboard

Displays all the information of every loan registered, and also gives direct access to management functions (renew loan, register devolution or delete loan record from database).

Loans filtered by user

Administrators can also filter loans by user by clicking user name on Loans management dashboard.


The system also provides access to 'administrators': they have access to every book, user or loan registered in the database. They can:

  • Manage books/authors/publishers (create, enable/disable, update, delete);
  • Manage registered users:
    • Enable / disable users;
    • Create books loans for them;
    • View every loan registered (also filtered by user);
    • Modify roles ('User' or 'Administrator');
    • Delete user from the database;
  • Manage loans (create, enable/disable, update, delete);

What did I use?

Backend

  • Spring framework
  • MySQL

Frontend

"Banquete San Jorge" - 2021

Last update: 17/12/21

This is a personal project for a family banquet business called "Banquete San Jorge". The website has three important sections:

  • "Sobre nosotros" (about us): this section tells the story of the family business, from the origin to the present.

  • "Solicitá un Presupuesto" (request a budget): this is for me the most important and useful feature of the website. It's a form that filters the key information that is required to elaborate a budget on a service request. After the form is completed, the information is sent via e-mail to the person in charge so he/she can elaborate a budget and send it to the client.

  • "Contacto" (contact): this section provides contact information and direct links to WhatsApp chats.

Carousel

This is the first section everyone sees.

Carousel

The second picture displays discounts and special prices.

Budget request form

Customers can provide the most important information about the event.

Budget request form

Customers can also specify what items they need.

Email sent via web form

This is the email that the person in charge gets, with all the information requested in the form.


In the last screenshot of the gallery you can see that the e-mail sent from the web form has also a design made with css. The extra features are the "WhatsApp" and "Maps" buttons:

  • The "WhatsApp" button redirects to the customer's chat;
  • The "Maps" button redirects to Google Maps, searching the location provided by the client in the form.

The goal of these features is to simplify the task of preparing a budget: you have the key information of the event, access to the location of the event and the customer's contact information all in one place.

  • Spring framework
  • MySQL

  • Thymeleaf
